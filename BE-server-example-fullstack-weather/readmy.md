1. Фронтенд (Клиентская часть)

- Интерфейс: Создать простую страницу (использовать React необязательно).
- Элементы управления: \* Одно поле ввода (input) для названия города. Одна кнопка для отправки.
- Логика: При нажатии на кнопку использовать функцию fetch() для отправки POST запроса на сервер.
- Передача данных: Отправить текст, введенный пользователем, в теле запроса (Body).
- Отображение: Полученный от сервера результат (строку с погодой) вывести на экран любым удобным способом.

2. Бэкенд (Серверная часть)

- Эндпоинт: Создать один маршрут: POST /weather.
- Прием данных: Когда приходит запрос, сервер должен прочитать название города из тела запроса (Request Body).
- Ответ сервера: Отправить в ответ текущую погоду в запрашиваемом городе в виде простой строки.
- Примечание: Информацию о погоде можно придумать самостоятельно («захардкодить»), подключать реальные API на этом этапе не обязательно.
- База данных: Выбрать интересные данные из запроса и ответа и сохранить их в базу данных LowDB.
