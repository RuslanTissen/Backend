1. Что такое Backend? (Основы)

Аналогия: Ресторан (eBay)

- Клиент (Frontend): Это посетитель в зале. Он видит красивое меню (интерфейс), но не может сам зайти на кухню и начать жарить котлеты.
- Сервер (Backend): Это кухня. Там хранятся продукты (данные) и повара их готовят. Посетитель не видит, что происходит за закрытыми дверями кухни, ему это и не нужно.
- API (Официант): Сервер всегда «ждет» (waits for request). Официант подходит к столу, принимает заказ (request), уносит его на кухню, а затем приносит готовое блюдо (response).

2. Живой пример: Работа с GitHub API

Давайте посмотрим на это с двух сторон, чтобы окончательно распутать этот «узел»:
С точки зрения Клиента (Frontend / Браузер):

-        Вы посылаете запрос (заказ) к GitHub API.
-        Вы получаете ответ (блюдо) обратно.

  С точки зрения Сервера (Backend / API):

-        Вы получаете запрос от клиента (официант принес чек на кухню).
-        Вы обрабатываете его и отправляете ответ назад.

  Важный нюанс: Серверы тоже могут быть «клиентами». Например, наша кухня может заказать продукты у другого поставщика (другого сервера).

3.  Кто есть кто в нашей системе?

- У нас всегда есть Клиент и Сервер.
- Fullstack разработчик — это человек, который настраивает и работу в зале (меню, столы), и процессы на кухне.
- Наш Сервер: Это приложение на NodeJS с использованием фреймворка Express.

  Наш Клиент: Обычно это браузер. НО! Клиентом может быть что угодно, что умеет отправлять запросы:

-        Терминал (cURL).
-        REST Client в VS Code (наш любимый инструмент).
-        Postman.
-        Даже само наше NodeJS приложение.

4.  Практический пример: Приложение погоды

Сейчас мы перейдем к коду и разберем наш проект «Погода». Но прежде два момента:

- LowDB: Помните, что это наша «учебная тетрадка» (fake database). Мы используем её для наглядности, чтобы понять, как данные записываются на диск.
- CORS (Cross Origin Resource Sharing): > Представьте, что вы пришли в один ресторан со своей едой из другого — вам могут запретить её есть.
- CORS — это правила безопасности. Когда ваш сайт пытается достучаться до другого сервера, браузер может заблокировать запрос из соображений безопасности.
