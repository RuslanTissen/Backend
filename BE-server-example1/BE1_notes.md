## Введение: что такое сервер

- Сервер — это программа
- Сервер — это не компьютер и не сайт.
- Это программа, которая постоянно работает в фоне.
- работает как сервис (daemon)
- запускается и живёт сама по себе
- не требует постоянного участия пользователя

## У сервера нет привычного интерфейса

- нет графического интерфейса (GUI)
- нет интерактивной командной строки
- мы не “кликаем” по серверу

# Сервер просто ждёт запросы и обрабатывает их.

## Как мы с ним взаимодействуем

- через запросы (HTTP, API)
- через другие программы (клиенты)
- иногда — через конфигурационные файлы

# Мы не управляем сервером напрямую —

мы задаём правила, по которым он работает.

## Клиент ↔ Сервер

- Клиент — браузер, мобильное приложение, другой сервер
- Сервер — принимает запросы и отдаёт ответы

# Клиент → Запрос → Сервер → Ответ → Клиент

сервер — это программа без интерфейса

## Сервер — это:

- программа, запущенная в фоне
- она слушает входящие запросы
- и отвечает на них по правилам (алгоритму)

## Что делает сервер?

- принимает HTTP-запросы
- обрабатывает их
- отправляет HTTP-ответы
  У сервера нет кнопок и окон
  Он работает через сеть и протоколы

## HTTP (Hyper Text Transfer Protocol), HTTPS

## Сервер на чистом Node.js

- сервер можно написать только на Node.js
- без библиотек и фреймворков
- это полезно для понимания, но редко используется на практике

## Почему не пишут на «чистом» Node?

- много низкоуровневого кода
- неудобная обработка маршрутов
- сложно масштабировать

## Инструменты для backend

- для этого существуют фреймворки
- мы будем использовать Express
  Express — это инструмент,
  который упрощает создание серверов

## Express и MERN

- Express — это E в MERN
- MongoDB
- Express
- React
- Node.js
  Express связывает Node.js и frontend

## Кодирование в реальном времени

- Создание приложения Node с помощью Express
- Конечная точка выглядит следующим образом
- "app.get(path, callback)" - В мире Node.js/Express функцию внутри app.get('/', ...) чаще называют Handler (обработчик) или Middleware.
- Иногда эти функции, обрабатывающие запросы, называются "контроллерами" = "controllers" (контроллер - устарело)

## Как работать с документацией (Гвоздь программы)

- документация — это единственный источник правды в 2025 году.
- Курсы на Stepik/Udemy 2022 года выпуска научат тебя app.get('\*'), который в Node v23+ просто вывалит ошибку.
- Лайфхак: Всегда смотри на вкладку Changelog или Migration Guide. Ошибка PathError из предыдущего урока — идеальное доказательство.

## Обновленный пример структуры (для Live Coding)

// routes.js
import { getUserHandler } from './handlers/user.js';
app.get('/user/:id', getUserHandler);

- Это разделение показывает, что роутинг — это «карта», а хендлер — «действие».
