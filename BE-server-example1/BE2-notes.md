### Инструментарий и автоматизация Backend-разработки

## Вспомним прошлый блок (Ретроспектива)

# Установка:

- npm install express
- `const app = express()`
- `app.get(<path>, <callback>)`
- `app.listen(<port>, <callback>)`
  Скелет сервера: Создали экземпляр app, настроили первый маршрут app.get и запустили прослушивание порта через app.listen.

Express-generator: Это "Create React App" для бэкенда. Полезно знать, но мы продолжаем писать код руками, чтобы понимать каждое действие.

1. Быстрые запросы через curl

- Инструмент командной строки, который можно использовать для создания запросов.
- Когда нужно проверить сервер, не обязательно открывать браузер (или Postman).
- команды:

```sh

  $ curl — это консольная утилита для отправки запросов.
  $ curl http://localhost:3000 — базовый GET-запрос.
  $ curl -v http://localhost:3000/pet — флаг -v == verbose, shows more info. Показывает «подкапотную» часть: заголовки (headers) и статус-коды. Это ваш главный инструмент отладки.
  $ curl --help all
  $ man curl

```

2. Автоматизация с nodemon

- Устали каждый раз нажимать Ctrl+C и заново писать node index.js после каждой запятой?
- nodemon (node monitor) — следит за изменениями в файлах и сам перезагружает сервер.
- Лайфхак: Настройте скрипт "dev": "nodemon index.js" в вашем package.json, чтобы запускаться одной командой npm run dev.

3. Внешние данные: Переходим на axios

- В браузере мы привыкли к fetch(). В Node.js старых версий его не было из коробки, но начиная с Node.js v18 (и окончательно в v20), fetch встроен в Node.js и не требует установки сторонних библиотек.
- Но Axios все равно стоит изучить, так как он остается стандартом в индустрии из-за удобных перехватчиков (interceptors), автоматической трансформации JSON и более понятной обработки ошибок.
- Зачем? Чтобы наш сервер мог сам «общаться» с другими серверами (запрашивать погоду, курсы валют или данные пользователей).

4. Работа с файлами: Модуль fs (FileSystem)

- Бэкенд — это про данные. Иногда данные лежат не в базе, а в простых файлах.
- Учимся читать текстовые (JSON-файлы).
- Разбираем, как превратить содержимое файла в ответ от вашего API.

## Практическая работа (Live Coding)

- Мы создадим приложение, которое объединяет всё вышеперечисленное:
- Запускается через nodemon.
- Использует axios для получения данных из интернета.
- Использует fs, чтобы сохранить эти данные в файл или прочитать их.
- Отдает результат в ответ на запрос через curl.
